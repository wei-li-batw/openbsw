Detailed Description

Cpp2Can Can Transceiver base interface

See also
AbstractCANTransceiver for a base HW independent implementation.

Statemachine

A subclass of ICanTransceiver has to implement the following statemachine. Initial state shall be CLOSED.

statemethod calledfollow state
CLOSEDinit()INITIALIZED
INITIALIZEDopen()OPEN
OPENclose()CLOSED
OPENmute()MUTED
MUTEDunmute()OPEN
MUTEDclose()CLOSED

◆ open() [2/2]

virtual ErrorCode can::ICanTransceiver::open ( CANFrame const &  frame)
pure virtual

Sets the transceiver to a status ready for transmission and reception under the restriction, that no other frame will be sent before frame frame. When the frame is transmitted, the state must be set to OPEN, e.g. in a tx callback function. If this functionality is not required, the function shall assert to inhibit unwanted usage.

◆ write() [1/2]

virtual ErrorCode can::ICanTransceiver::write ( CANFrame const &  frame)
pure virtual

Writes a CANFrame to Can bus

Parameters
frameframe to transmit
Returns
  • CAN_ERR_OK: transmission was successful
  • CAN_ERR_TX_HW_QUEUE_FULL: tx-hardware-queue is full
  • CAN_ERR_TX_OFFLINE: transceiver was offline while trying to transmit
  • CAN_ERR_TX_FAIL: transmission failed with unspecified error
  • CAN_ERR_ILLEGAL_STATE: method was called in wrong state
Precondition
getState() == INITIALIZED || OPEN

Timing

A call to this method is non-blocking, i.e. it immediately returns an error if e.g. the hw-queue is full and relies on the calling task to retry the transmission.

Implemented in can::SocketCanTransceiver, and bios::CanFlex2Transceiver.
